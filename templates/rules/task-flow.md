# 任務流程規則

> 收到任何任務時，CLAUDE.md 會指向這裡。這些步驟確保 agent 在動手前先搞清楚狀況。

## 收到任務時

1. **解析指令中的名詞**：使用者提到的專案名、檔案名、工具名，都是線索。列出來
2. **查看現場**：對每個名詞，先 `ls` 對應路徑 → 有檔案就 `Read`（README 優先）→ 理解現有結構
3. **確認「是新增還是修改」**：現場有東西 = 修改模式；現場是空的 = 新增模式
4. **不確定就問**：任何拿不準的地方，直接問使用者
5. **再動手**：完成以上步驟才能執行寫入操作

## 執行任務時

- 遇到不確定的地方，直接問，不要自作主張
- 如果任務太大，建議拆分成多個小任務
- 保持進度可見：完成一個階段就告訴使用者

## 任務完成驗證

任務完成前，依任務類型執行對應驗證：

- **寫入檔案** → `ls` 確認檔案在正確位置且大小合理
- **技術任務** → 跑測試或實際執行確認結果
- **資訊查詢** → 交叉驗證來源

未通過驗證 = 任務未完成。

## 檔案存放原則

- 所有檔案一律存放在 workspace 底下，不要散落在其他地方
- 每個主要項目放在 `projects/` 下，各自有獨立的資料夾
- 臨時性產出放在 `outputs/`
